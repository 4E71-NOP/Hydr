« utilisateur » : Représente un utilisateur.

Les arguments associés à l'élément « utilisateur »
nom Champ libre.
login Champ libre (doit etre unique pour le site).
password Champ libre.
status INACTIF, ACTIF, SUPPRIME
droit_forum OUI / NON
email Champ libre.
msn Champ libre.
aim Champ libre.
Icq Champ libre.
yim Champ libre.
website Champ libre.
perso_nom Champ libre.
perso_pays Champ libre.
perso_ville Champ libre.
perso_occupation Champ libre.
perso_interet Champ libre.
lang Définit la préférence utilisateur pour la langue.
pref_theme Définit la préférence utilisateur pour le theme.
pref_newsletter OUI / NON
pref_montre_email OUI / NON
pref_montre_status_online OUI / NON
pref_notification_reponse_forum OUI / NON
pref_notification_nouveau_pm OUI / NON
pref_autorise_bbcode OUI / NON
pref_autorise_html OUI / NON
pref_autorise_smilies OUI / NON
image_avatar Champ de type ficher. Défini l'icone.
admin_commentaire Champ libre.
filtre Champ libre. Permet de filter le résultat de la commande "show" sur le nom interne des éléments cible.

Exemple:

> site_context "MultiWeb_Engine";
> add user login "dev_senior" perso_name "Dev senior" password "drowssap" status "ACTIVE" image_avatar "../websites-datas/00_mwm/data/images/avatars/public/dieu.gif";
> user dev_senior join_group Developpeurs_senior primary_group OUI;
